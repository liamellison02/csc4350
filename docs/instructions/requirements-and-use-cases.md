# Assignment: Project Description, Use Case Diagram and Requirements

1. Create a Use Case Diagram for your system:
   1. Should specify each user role
   2. Set of use cases (titles only) or tasks for each role
2. List the functional and non-functional requirements for your application
   1. See https://jelvix.com/blog/software-requirements-specification and the
      attached file (SRS 4.0) for more examples of requirements
3. Based on your requirements decide the architecture of your application. What
   technologies will you use for your front end, back end and database?
