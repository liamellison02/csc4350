# Assignment 6: Wireframes, Test Scenarios, and Test Cases

## Instructions

Draw wireframes or mockups of each of your screens in your software.
You can use the drawing software of your choice (Google Docs/Slides Powerpoint,Figma,MS Paint etc) to create each wireframe.
You do not need to have one wireframe per testing scenario.

Write at least four testing scenarios for your system, which describes a user completing a task in your software.
Examples of scenarios are below for a flight booking system.

1. User inputs departure airport and chooses to see only direct flights. User quits.

2. User inputs departure airport and chooses to see all flights. User quits.

3. User chooses destination country and chooses to see all flights. User quits.

4. User inputs departure airport and chooses to see direct flights. User sets filter specifying departure times and prices. User quits.

5. User inputs departure airport and chooses to see direct flights. User sets filter specifying departure times and prices. User selects a displayed flight and clicks through to airline website. User returns to holiday planner after booking flight.

For each of the test scenarios you've created, create at least one unit test using the appropriate testing library for your application.
Testing libraries and documentation are written below.
These tests should be pushed to your git hub repository.
Submit a link to your GitHub repository and submit screenshots with the results of running your tests.
Each team member's sprint report should mention at least one commit of each team member contributing to a unit test.
